# 6G Skill-Based Agentic Core Network (Agentic Layer)

This project is a prototype for a decoupled AI layer built on top of a 5G core (e.g., `free5gc` and `ueransim-go`). It implements a **Skill-Based Agentic Architecture** for 6G, enabling intent-driven networking by transforming the network into a dynamic "Skill Registry."

## Project Overview

- **Objective:** Bridge the gap between abstract user intents (natural language) and deterministic network signaling (3GPP OpenAPI).
- **Core Reference:** 3GPP TSG SA WG2 Meeting #173 (S2-2600222) - KI#18, KI#19-Solution of Skill-Based Agentic Architecture for Semantic Discovery.
- **Key Concepts:**
    - **Intent (The "Why"):** Abstract goals generated by users or applications.
    - **Skill (The "Interface"):** Standardized URIs (e.g., `mcp://skill/...`) that define executable capabilities.
    - **Service Directive (The "How"):** Deterministic 3GPP Service Operations executed by target nodes.

## Microservices Architecture

The system is structured into three primary Go-based microservices:

1. **AAIHF (Agentic AI Host Function):** The reasoning brain. Processes intents and discovers skills using the `google/adk-go` framework. Supports real-time SSE streaming on `/stream`.
2. **ACRF (Agentic Capability Repository Function):** A dynamic skill registry that stores "Agentic Skill Profiles" and maps them to service endpoints. Uses semantic embeddings for discovery.
3. **A-IGW (Interworking Gateway):** The "Universal Translator" that translates skill invocations into 3GPP API calls. Uses **Temporal.io** for stateful, reliable execution.

## Dependencies

- **Go 1.22+**
- **Temporal.io Server**: Required for A-IGW workflow orchestration.
- **LLM Providers**: Supports `gemini-1.5-flash` and `kimi-k2.5` (Moonshot).

## Testing and Verification (The "Test Wall")

The project implements a multi-layered testing strategy:

- **BDD Tests**: Executable scenarios in `tests/features/` using `godog`.
- **Temporal Workflow Tests**: Isolated workflow verification in `internal/translator/temporal_skills/`.
- **System Integration Tests**: End-to-end service plumbing in `tests/system_integration_test.go`.
- **Unit Tests**: Component-level tests (e.g., `internal/agent/tools_test.go`).

Run all tests:
```bash
go test -v ./...
```

## Running the System

1. **Start Temporal Server**:
   ```bash
   temporal server start-dev --ip 127.0.0.1
   ```
2. **Start Services**:
   ```bash
   ./scripts/run_services.sh
   ```
3. **Verify System**:
   ```bash
   ./scripts/verify_system.sh
   ```

## Development Conventions

- **Environment Variables**: MUST use the `AGENTIC_` prefix (e.g., `AGENTIC_TEMPORAL_HOST`).
- **OpenSpec Workflow**: Use `/opsx` commands for all feature development.
- **Separation of Concerns**: AI reasoning (AAIHF) is strictly decoupled from deterministic 3GPP state machines (IGW).
- **Observability**: AAIHF emits real-time events via SSE for reasoning visualization.
