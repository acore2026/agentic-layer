# 6G Skill-Based Agentic Core Network (Agentic Layer)

This project is a prototype for a decoupled AI layer built on top of a 5G core (e.g., `free5gc` and `ueransim-go`). It implements a **Skill-Based Agentic Architecture** for 6G, enabling intent-driven networking by transforming the network into a dynamic "Skill Registry."

## Project Overview

- **Objective:** Bridge the gap between abstract user intents (natural language) and deterministic network signaling (3GPP OpenAPI).
- **Core Reference:** 3GPP TSG SA WG2 Meeting #173 (S2-2600222) - KI#18, KI#19-Solution of Skill-Based Agentic Architecture for Semantic Discovery.
- **Key Concepts:**
    - **Intent (The "Why"):** Abstract goals generated by users or applications.
    - **Skill (The "Interface"):** Standardized URIs (e.g., `mcp://skill/device/fleet-update`) that define executable capabilities.
    - **Service Directive (The "How"):** Deterministic 3GPP Service Operations executed by target nodes.

## Microservices Architecture

The system is structured into three primary Go-based microservices:

1. **AAIHF (Agentic AI Host Function):** The reasoning brain. Processes intents and discovers skills using the `google/adk-go` framework.
2. **ACRF (Agentic Capability Repository Function):** A dynamic skill registry that stores "Agentic Skill Profiles" and maps them to service endpoints.
3. **A-IGW (Interworking Gateway):** The "Universal Translator" that receives skill invocations and translates them into sequences of 3GPP API calls to the core network.

## Testing and Verification

The project includes both automated integration tests and a manual verification script.

- **Automated Tests:**
  ```bash
  go test -v ./tests/...
  ```
- **Manual Verification Script:**
  Starts all services, sends a test intent, and prints the result.
  ```bash
  ./scripts/verify_system.sh
  ```
- **Test-First Policy:** All new features or bug fixes MUST be verified by running the existing integration tests (or adding new ones) before implementation is considered complete.

## Development Workflow: OpenSpec

This project uses the **OpenSpec** workflow for spec-driven development.
- **Workflow Tools:** Located in `.gemini/commands/opsx/` (e.g., `propose`, `apply`, `explore`).
- **Configuration:** `openspec/config.yaml` defines the project-specific rules and tech stack.
- **Specs & Changes:** Artifacts are managed under `openspec/specs/` and `openspec/changes/`.

## Building and Running

- **Environment:** Requires Go 1.22+ and access to a 5G core (free5gc/ueransim).
- **Initialization:**
  ```bash
  go mod init github.com/google/6g-agentic-core
  go mod tidy
  ```
- **Execution:**
  - **ACRF:** `go run cmd/acrf/main.go` (Port 18080)
  - **IGW:** `go run cmd/igw-fleet/main.go` (Port 18081)
  - **AAIHF:** `go run cmd/aaihf/main.go` (Port 18082 - requires `AGENTIC_GEMINI_API_KEY`)

## Development Conventions

- **Language:** Idiomatic Go.
- **API Style:** REST/HTTP for communication between microservices.
- **Discovery:** Skills must be registered with ACRF before invocation.
- **Separation of Concerns:** Keep AI reasoning logic (AAIHF) strictly decoupled from deterministic 3GPP state machines (IGW).
